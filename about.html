What is CollectionBuilder?

CollectionBuilder is an open source tool for creating digital collection and exhibit websites driven by metadata, and powered by modern static web technology that works through two basic steps:

  1. The user provides a well structured metadata spreadsheet and a directory of digital files,
  2. CollectionBuilder processes the metadata to generate a variety of visualizations--surfacing the unique facets and features of the content, creating interactive pathways to browse and discover items while providing overall context.

Each visualization is configurable using a few basic variables or config files.

Each is a self contained static web page that can be hosted on the most basic web servers without the need for a database or server-side processing language, enabling simple security and speedy performance.

Check out our Docs or our Workshop Materials for more information.

Current projects

  • CollectionBuilder-GH - A simple template for hands-on teaching about digital libraries, designed to be hosted on GitHub Pages. It can be used in a workshop setting to take participants through digitization and metadata creation, to having a live collection site for free. An static alternative to teaching with full scale DAMS such as Omeka.
  • CollectionBuilder-Skin for CONTENTdm - in-development version of CollectionBuilder used by the University of Idaho Library to create "skins" for the digital library collections they maintain via a hosted CONTENTdm instance. Will be developed into a generic CONTENTdm "skin" that can be adopted by others.
